## рҹ”„ Component Lifecycle

### Component Mount
```
1. Component renders
2. Hook calls context
3. Context provides cached/initial value
4. Component displays data
5. If loading, show spinner
6. When data arrives, update display
```

### Component Update
```
1. Context value changes
2. Hook detects change
3. Component re-renders
4. Memoized values prevent unnecessary work
5. Only affected parts update
```

## рҹҡҖ Performance Optimizations

### Memoization Strategy
```
ThemeContext
в”ңв”Җ useMemo: isDark
в”ңв”Җ useMemo: colors
в””в”Җ useMemo: contextValue
    вҶ“
useThemeColors
в””в”Җ Returns memoized colors
    вҶ“
Component
в””в”Җ Only re-renders when colors actually change
```

### Caching Strategy
```
useMedicines
в”ңв”Җ useRef: cacheRef
в”ңв”Җ Check cache validity (30s)
в”ңв”Җ Return cached if valid
в””в”Җ Fetch fresh if invalid
    вҶ“
AppDataContext
в””в”Җ Provides cached data to all consumers
    вҶ“
Multiple Components
в””в”Җ Share same cached data
```

## рҹ“ҡ File Structure

```
/lib/context/
в”ңв”Җв”Җ AppContext.tsx          # Main orchestrator
в”ңв”Җв”Җ ThemeContext.tsx        # Theme management
в”ңв”Җв”Җ UserContext.tsx         # User data
в””в”Җв”Җ AppDataContext.tsx      # App data

/lib/hooks/
в”ңв”Җв”Җ useThemeColors.ts       # Theme utility hooks
в”ңв”Җв”Җ useMedicines.ts         # Medicine data hooks
в”ңв”Җв”Җ useDoses.ts             # Dose data hooks
в””в”Җв”Җ index.ts                # Centralized exports

/components/
в”ңв”Җв”Җ /ui/                    # Use useThemeColors()
в””в”Җв”Җ /medicine/              # Use useThemeColors()

/app/
в”ңв”Җв”Җ _layout.tsx             # AppProvider wrapper
в””в”Җв”Җ (tabs)/                 # Use all hooks
```

---

**Key Takeaways:**

1. **Nested Providers**: Context providers wrap each other for layered access
2. **Specialized Hooks**: Use specific hooks (useThemeColors) over general ones (useTheme)
3. **Memoization**: Values memoized at context level for performance
4. **Caching**: Data cached for 30s to reduce database queries
5. **Type Safety**: Full TypeScript support throughout the chain
6. **Single Source**: All state managed in one place, accessed anywhere
